# Abstract

Large language model (LLM) inference and parameter-efficient personalization increasingly rely on GPU acceleration, yet GPUs see the full activation transcript: prompt-derived hidden states, KV cache entries, LoRA adapter factors, gradient tensors, and the generation trace. None of these are public, and prior literature shows that each is sufficient on its own to recover sensitive user data. A naive trusted-execution-environment (TEE) deployment that runs the entire model inside the TEE is impractical at LLM scale; conversely, a naive GPU-offload deployment exposes everything we want to protect.

This paper presents an artifact-backed prototype of **operator-compatible masked execution** for generative LLM inference and LoRA personalization. A trusted-side controller (emulated locally in our prototype, *not* a real TEE) samples fresh masks each call, applies them at Linear boundaries on the right of the activation tensor so that decoder-only generation, KV-cache append, RoPE, and grouped-query attention remain compatible, and offloads dense matrix multiplication to an untrusted GPU. Nonlinear layers are surrounded by **operator-compatible nonlinear islands**: paired channel permutations for GELU/ReLU and SwiGLU, orthogonal masks for RMSNorm, and mean-preserving orthogonal masks for LayerNorm; each island is sandwiched by dense Linear masks and protected by a boundary pad. The same machinery extends to a **private LoRA training path** — masked LoRA forward, masked LoRA backward, rank padding with stronger dummy distributions, and a multi-layer training step — while loss and optimizer remain on the trusted side. A constant-time-emulated training mode (cost-model proxy only) flattens a worst-case timing classifier.

We evaluate the prototype with: (i) empirical correctness equality versus a plain reference for GPT-2 and a modern decoder-only wrapper (RMSNorm + SwiGLU + RoPE + GQA), including prefill, decode_step, and greedy generation; (ii) adaptive proxy attackers — ridge / MLP inverters, signature / Sinkhorn permutation recovery, linkability AUC, LoRA adapter and gradient extraction, spectral-cliff and ensemble rank inference, and a cost-model timing classifier; and (iii) a local-emulation wall-clock benchmark using `time.perf_counter`. The artifact registry covers 22 outputs (11 inference, 11 LoRA), all consolidated into paper-side CSV / Markdown / LaTeX tables, seven figures, and a `claims_mapping` document that pins every paper sentence to one of `supported` (8), `proxy_supported` (5), or `unsupported` (8).

This is a **prototype with proxy-evaluated security**. We make no formal, cryptographic, or semantic security claim; we do not report real TEE wall-time; we do not implement full Qwen / TinyLlama / LLaMA LoRA fine-tuning; we do not integrate PEFT / DeepSpeed / vLLM / FlashAttention; we do not claim the padded LoRA rank itself is hidden; loss and optimizer remain trusted-side. Within these boundaries, our results show that generation-compatible right masking, operator-compatible nonlinear islands, and a private LoRA training path produce a transcript on which the strongest proxy attackers we evaluate are kept close to random chance, while the wrapper reproduces the plain reference output token-for-token in the tested configurations.
