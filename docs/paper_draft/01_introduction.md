# Introduction

## Motivation

Large language model inference is increasingly executed on accelerators whose host environment is not within the user's trust boundary. Even when the model weights themselves are public — which is now the common case for open-weight Qwen / LLaMA-family decoders — the *runtime tensors* that flow through the forward pass remain user-derived: the tokenised prompt, the per-layer hidden states, the key/value cache contents, the output logits, and (when applicable) the contribution of a private LoRA adapter trained on confidential user data. A model-serving provider that loads public weights onto an untrusted accelerator therefore still needs to keep these *user-derived* tensors out of the accelerator's plaintext view, even though the weights themselves require no protection.

Decoder-only generation makes this harder than the textbook outsourced-matmul setting. The forward pass interleaves linear projections (Q/K/V, o, gate, up, down, LM head) with structure-sensitive operators that are *not* generic matmuls: RMSNorm normalises by per-row L2 norm, rotary positional embedding (RoPE) rotates each `(j, j + d_h/2)` channel pair, grouped-query attention computes a softmax over a score matrix `Q K^T`, SwiGLU multiplies element-wise across an MLP intermediate, and the KV cache *appends* per-token entries across many decode steps. On top of this, real serving systems use paged KV memory, sliding-window attention for long context, multi-session batching, and a stack of generation-time logit processors (temperature, top-k, top-p, repetition penalty, stop tokens, bad words, forced tokens). Each of these operators has its own algebraic invariant that an obfuscation mask must respect or break.

## Problem

We need a protocol that (i) uses the accelerator for the dense matmul work, (ii) limits accelerator-visible plaintext to a small, *declared* set of leakage surfaces, (iii) preserves end-to-end greedy / sampling equivalence with plaintext execution, and (iv) keeps the online trusted-controller / accelerator boundary down to as few round trips per decode step as possible — ideally one. A naïve approach — running every nonlinear operator inside the trusted side and only sending matmul work to the accelerator — defeats the bandwidth advantage of the accelerator and yields `O(L)` round trips per decode step. Conversely, masking with arbitrary dense randomness through the nonlinear cores breaks RMSNorm (the row L2 norm changes), RoPE (the 2D rotation does not commute with a generic mask), softmax (`softmax(S + R) ≠ softmax(S)` unless `R` is row-constant), and SwiGLU (element-wise nonlinearity does not absorb generic masks). The design problem is therefore *not* "mask everything"; it is "find, for each operator, the *family* of masks that operator commutes with, and pay the resulting cost as a declared mode knob rather than as silent leakage".

## Key insight

We use **operator-compatible mask families** instead of operator-agnostic randomness. Each operator is paired with the mask family it commutes with at the algebraic level:

* RMSNorm pairs with row-wise *orthogonal* masks `Q` (the per-row L2 norm is preserved iff `Q Q^T = I`).
* RoPE pairs with *block-diagonal 2D rotations* aligned to the RoPE planes `(j, j + d_h/2)` (the 2D rotation commutes with the RoPE rotation within the same plane).
* GQA / MQA softmax pairs with Q / K masks satisfying `B_Q B_K^T = I` per query head; the score matrix `Q_tilde K_tilde^T` then equals `Q K^T` exactly.
* SwiGLU pairs with a *shared paired permutation* `P` across the gate and up branches; the down projection absorbs `P^{-1}`.
* KV cache append pairs with per-(layer, head) right masks `N_K`, `N_V` that are shared across all positions of a session.
* LM head pairs with an orthogonal vocab mask `N_vocab` (for tiny vocab) or a scalable permutation / block-diagonal variant (for real LLM vocab).

The pad enters *only* at the linear boundaries: a fresh additive pad `T` is applied to make `x_tilde = (x - T) M`, the next linear absorbs the pad via a precomputed `C = T W N`, and the next nonlinear operator therefore sees a *clean* transformed activation, never a pad-contaminated one. We track per call and per mode the four flags `pad_enters_rmsnorm_core`, `pad_enters_rope_core`, `pad_enters_swiglu_core`, `pad_enters_softmax`, all of which must remain `false` in the main protocol (and are checked by the test suite). The price of operator-compatibility — what *cannot* be made invariant — is paid as a declared mode knob: norm-mask *granularity* trades off Gram leakage against per-token transition cost, attention-map *privacy mode* trades off attention-map hiding against the one-round-trip property, LM-head *masking choice* trades off scalability against the strength of vocab-index hiding.

A boundary-transition trick keeps the trusted side off the online path. The trusted side compiles, *per call*, transition tables `A_i = Q_i^{-1} M_i` and `C_i = T_i M_i` for every linear boundary; the accelerator then transitions from a compatible state `H_l Q_l` to a padded-boundary state `(H_l - T) M` *on the accelerator* without re-entering the TEE between layers. In the exact main mode the entire forward proceeds with `intermediate_tee_reentry = false` and `online_boundary_round_trips_per_decode_step = 1`.

## Contributions

* **C1.** A low-interaction padded masked execution protocol for public-weight decoder-only LLMs. Fresh linear-boundary pads and operator-compatible masks together yield exact greedy / sampling equivalence with plaintext at one online TEE-accelerator round trip per decode step in the main mode, with no intermediate TEE re-entry. Validated under CPU local emulation on a synthetic tiny modern-decoder surrogate that mirrors the LLaMA / Qwen forward graph; greedy match `1.0`, `lm_head_recovery_max_abs_error` at float64 machine precision (`< 1e-14`).

* **C2.** A decoder-operator coverage framework. We pair each mainstream decoder operator with the mask family it commutes with: RMSNorm with sequence / chunk / token granularity (`outputs/norm_granularity_low_interaction.json`); standard 1D RoPE with block-diagonal rotation aligned to RoPE planes, eliminating any plaintext Q / K / V transient on the accelerator (`outputs/modern_decoder_rope_safe_low_interaction.json`); GQA / MQA softmax via the QK invariant; SwiGLU via paired permutation; KV cache via per-(layer, head) right masks under standard, paged, and sliding-window storage layouts; the LM head via dense / permutation / block-diagonal variants with declared scalability trade-offs (`outputs/lm_head_scalability.json`); LoRA inference at every supported insertion site (`outputs/lora_protocol_integration.json`); and trusted-side generation processors covering greedy, temperature, top-k, top-p, repetition penalty, stop / bad-words / forced tokens (`outputs/generation_processor_coverage.json`).

* **C3.** A CPU local-emulation experiment suite and per-claim audit. The suite (`outputs/paper_experiment_suite.json`) aggregates 16 sub-experiments into a paper-safe report. The audit (`outputs/paper_claims_audit_v2.json`) classifies 26 claims into 15 supported, 1 proxy-supported, and 10 explicitly unsupported, each with a safe wording, an unsafe wording, an evidence artifact path, and a remaining blocker. A precision sweep (`outputs/precision_quantization_stability.json`) reports per-mode error under fp32 / bf16 / fp16 / int8 / int4 simulations and recommends well-conditioned mask families for low precision. A coverage audit (`outputs/decoder_component_coverage_audit.json`) enumerates 10 components covered in the main protocol, 6 partially covered / extensions, and 9 explicitly unsupported (M-RoPE, MoE, MLA, speculative decoding, real vLLM / FlashAttention backend, real hardware side channels, full active malicious security, LoRA training, full Qwen / LLaMA deployment).

## Scope and honest non-claims

This work is a CPU local-emulation prototype. We do *not* measure real GPU or TEE wall-clock; the cost-model evidence (`outputs/paper_cost_model.json`) is symbolic plus tiny / real-config FLOP and storage estimates. We do *not* claim formal cryptographic, semantic, or differential-privacy security; the protocol is operator-compatible algebraic obfuscation, not a cryptographic protocol. We do *not* evaluate hardware side channels; the window-size policy and the KV cache length are public by construction. The active-adversary integrity prototype is a probabilistic spot-check (`outputs/integrity_spotcheck.json`), not verifiable computation. We do *not* support M-RoPE, MoE routing, Multi-Head Latent Attention, speculative decoding, or LoRA backward training, and we do *not* claim full Qwen / LLaMA deployment; each of these is marked unsupported with a stated remaining blocker. These non-claims are not buried in the limitations — they are enumerated in the claim audit and gated by automated tests.
